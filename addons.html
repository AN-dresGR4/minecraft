Guía para creadores de addons

Si estás interesado en aportar al proyecto con tu creatividad y tu capacidad de lógica en programación, puedes ayudar creando addons para MSP.
Los addons son funciones extras basadas en archivos de texto, que ejecutan una serie de comandos fáciles de entender.
Si hiciste un addon y quieres que sea exhibido, puedes abrir un ticket en en el Discord pidiendo la verificación de tu addon.
Si tu addon es aceptado recibirás un rol especial (Los mejores addons serán añadidos en futuras versiones de MSP)
Si tienes dudas o problemas con la creación de addons, puedes comentarlo en el canal de Discord reservado para esto y esperar a que alguien te ayude o te explique.

1. Secciones
Las secciones son pequeños bloques que contienen datos adentro (o comandos). Debe iniciar en algún momento, y su finalización es cuando otro bloque se inicie. Interpretar una sección es fácil pues empieza con : La síntaxis para iniciar una sección sería 
:seccion
1.1 Sección principal
Al inicio de cualquier addon, debe iniciarse (obligatoriamente) la sección principal, esta sección va a contener los datos de tu addon (funcionan para distinguir tu addon de otros). Se crea escribiendo esto al inicio de tu addon: 
:main
(Nota: Todas las propiedades de la sección principal deben empezar con a_) Luego, abajo de eso irán los siguientes datos Nombre del addon Versión del addon Autor del addon Para especificar estas propiedades usaremos 
a_name <nombre del addon>
a_version <version del addon>
a_author <autor del addon>
Si usamos datos de prueba, el addon quedaría algo así 
:main
a_name Mi addon de prueba
a_version 1.0
a_author Elyx
1.2 Sección: Botones
Una vez hayamos declarado la sección principal y sus propiedades, abajo de ella podremos declarar botones (son los que el usuario va a poder elegir) Los botones usan un identificador (el id lo usa MSP para saber qué acciones registrar) la síntaxis para crear un botón sería esta: 
:button <id del botón>
(Nota: Los id's de los botones no pueden contener espacios ni ser idénticos a los id's de otros botones del mismo addon, sin embargo, pueden ser lo que tu quieras) Usando la propiedad text debemos especificar el texto que va a salir en el botón. Un ejemplo completo de un botón visible para el usuario sería este: 
:button boton_de_prueba
text Elige mi botón de prueba!!
2. Comandos
Ahora que sabes cómo hacer secciones y darles propiedades, veremos los comandos básicos y su uso. Los comandos son las acciones que debe realizar un botón al ser elegido. Algunos comandos no necesitan de argumentos extra para funcionar, otros sí, la síntaxis de un comando sería esta: 
comando [argumento1] [argumento2] [etc...]
Un ejemplo de un addon con lo que llevamos hasta ahora sería este: 

:main
a_name Mi addon de prueba
a_version 1.0
a_author Elyx

:button boton_de_prueba
text Elige mi botón de prueba!!
comando1
comando2
etc..
Say's
say 
nsay 
csay
Say: Dice algo de manera colorida y decorada con un [+]
Nsay: Dice algo de manera colorida pero sin el decorador
Csay: Dice algo sin color y sin decorador
un ejemplo de uso sería este: 
...
:button boton_de_prueba
text Elige mi botón de prueba!!

say Colorido y con decorador
nsay Solo colorido
csay Sin color y sin decorador
cls
Este comando no requiere argumentos. Sirve para limpiar la pantalla. Sintaxis: 
cls
un ejemplo de uso sería este: 
...
:button boton_de_prueba
text Elige mi botón de prueba!!

say Este texto se verá antes de limpiarse la pantalla
cls
say Este texto se verá después de limpiarse la pantalla
pause
Este comando pide que el usuario aprete enter antes de seguir con el resto de comandos. No requiere argumentos Sintaxis: 
pause

un ejemplo de uso sería este:

...
:button boton_de_prueba
text Elige mi botón de prueba!!

say Antes
pause
say Después
pause
sleep

Este comando hace que el addon espere cierta cantidad de segundos antes de seguir con el resto de comandos. Requiere el argumento segundos Sintaxis:

sleep <cantidad de segundos>

un ejemplo de uso sería este:

...
:button boton_de_prueba
text Elige mi botón de prueba!!
say Verán este texto y esperarán 3 segundos
sleep 3
say Este texto se verá después de haber pasado 3 segundos
pause
remove_server

Este comando elimina el servidor actual del usuario (si existe). No requiere argumentos Sintaxis:

remove_server
un ejemplo de uso sería este: 
...
:button boton_de_prueba
text Elige mi botón de prueba!!
say Eliminando servidor...
remove_server
cls
say Servidor eliminado!
pause
